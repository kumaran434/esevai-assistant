# பிழை கண்காணிப்பு மற்றும் சரிபார்ப்பு பட்டியல் (Error Tracking & Verification)

இந்தக் கோப்பு இந்த செயலியில் இதுவரை கண்டறியப்பட்ட பிழைகள் மற்றும் அவற்றிற்கான நிரந்தரத் தீர்வுகளைக் கண்காணிக்கப் பயன்படுகிறது. ஒவ்வொரு முறையும் புதிய மாற்றங்கள் செய்யப்படும்போது, இந்தப் பட்டியலில் உள்ள பிழைகள் மீண்டும் வராமல் இருப்பதை உறுதி செய்யவும்.

## கண்டறியப்பட்ட முக்கியமான பிழைகள் (Logged Errors)

### 1. Electron Preload Script Syntax Error
- **பிழை:** `SyntaxError: Unexpected identifier '$'`
- **காரணம்:** `electron-main.cjs` கோப்பில் உள்ள `preloadContent` சரத்திற்குள் (String) ES6 Template Literals (`${}`) பயன்படுத்தப்பட்டது. இது சில சமயங்களில் பிரவுசர் வியூக்குள் (BrowserView) நெஸ்டட் சரமாக (Nested String) இருக்கும்போது பிழையைத் தருகிறது.
- **தீர்வு:** Template literals-க்குப் பதிலாக சாதாரண String Concatenation (`+`) பயன்படுத்தப்பட்டது.
- **நிலை:** ✅ சரி செய்யப்பட்டது.

### 2. AI Scanning Timeout
- **பிழை:** AI பகுப்பாய்வு செய்ய அதிக நேரம் எடுத்து 'Scanning' நிலையிலேயே நின்றுவிடுதல்.
- **காரணம்:** இணையதளத்தின் அளவு பெரியதாக இருக்கும்போது (Complex DOM) 50 வினாடிகள் என்ற கால அளவு போதுமானதாக இல்லை.
- **தீர்வு:** 'Scanning Timeout' 90 வினாடிகளாக அதிகரிக்கப்பட்டது மற்றும் செயல்பாட்டின் நிலை குறித்து கூடுதல் தொழில்நுட்பத் தகவல்கள் (Technical Logs) சேர்க்கப்பட்டன.
- **நிலை:** ✅ சரி செய்யப்பட்டது.

### 3. Sidebar Resizing Issue over BrowserView
- **பிழை:** எலக்ட்ரான் பிரவுசர் வியூ (Electron BrowserView) மீது மவுஸை நகர்த்தும்போது சைடுபார் அகலத்தை மாற்ற முடியவில்லை (Mouse events blocking).
- **காரணம்:** BrowserView ஒரு தனி விண்டோவாக இருப்பதால், மெயின் விண்டோவில் உள்ள மவுஸ் ஈவென்ட்களை அது தடுத்துவிடும்.
- **தீர்வு:** ரீசைஸிங் (Resizing) செய்யும்போது ஒரு வெளிப்படையான 'Overlay Div' சேர்க்கப்பட்டது.
- **நிலை:** ✅ சரி செய்யப்பட்டது.

- [ ] புதிய மாற்றங்களைச் செய்த பிறகு `npm run build` மூலம் கம்பைல் செய்து பார்க்கவும்.
- [ ] `electron-main.cjs` கோப்பில் எஸ்கேப் எழுத்துக்கள் (Backticks) சரியாக உள்ளனவா என்று சரிபார்க்கவும்.

### 4. Module Not Found / ESM Conflict / Path Error (Production Build)
- **பிழை:** `TypeError: Cannot read properties of undefined (reading 'join')` மற்றும் `ReferenceError: require is not defined`.
- **காரணம்:** 
  1. `package.json`-இல் `"type": "module"` இருப்பதால் ஏற்பட்ட ESM/CommonJS மோதல்.
  2. `path` மாட்யூல் `electron`-இன் ஒரு பகுதி எனத் தவறாகக் கருதப்பட்டு `const { path } = require('electron')` என அழைக்கப்பட்டிருந்தது. உண்மையில் `path` என்பது ஒரு தனி Node.js மாட்யூல்.
- **தீர்வு:** 
  1. `main-process/` போல்டரில் உள்ள அனைத்துக் கோப்புகளின் நீட்டிப்புகளும் (Extensions) `.js`-இல் இருந்து `.cjs`-க்கு மாற்றப்பட்டன (இது CommonJS முறையை எலக்ட்ரான் சூழலில் உறுதி செய்யும்).
  2. அனைத்துக் கோப்புகளிலும் `const path = require('path');` என்பது முறையாகத் தனித்தனியாகச் சேர்க்கப்பட்டது.
  3. `electron-main.cjs`-இல் `require` பாதைகள் அனைத்தும் `.cjs` கோப்புகளை நோக்கியே இருக்குமாறு மாற்றப்பட்டன.
- **ஆடிட் (Audit):** அனைத்து மெயின் ப்ராசஸ் கோப்புகளும் (`electron-main.cjs`, `automation-logic.cjs`, `ipc-handlers.cjs`, `session-manager.cjs`, `utils.cjs`) ஒவ்வொன்றாகச் சரிபார்க்கப்பட்டன. இறக்குமதிகள் (Imports) மற்றும் எக்ஸ்போர்ட்கள் (Exports) இப்போது முற்றிலும் சரியாக உள்ளன.
- **நிலை:** ✅ **நிரந்தரமாகச் சரி செய்யப்பட்டது.** 

### 5. Automation: Address Splitting & Language Conflict
- **பிழை:** முகவரி பிரிக்கப்படாமல் ஒரே இடத்தில் நிரப்பப்படுதல் மற்றும் தமிழ்/ஆங்கில புலங்கள் சரியாக அடையாளம் காணப்படாமை.
- **காரணம்:** AI Prompt-இல் முகவரி நுணுக்கங்கள் குறைவாக இருந்தது மற்றும் UI மேப்பிங் தர்க்கம் முகவரியைப் பிரிப்பதில் பலவீனமாக இருந்தது.
- **தீர்வு:** 
  1. `geminiService.ts`-இல் முகவரியைத் தனித்தனியே (doorNo, street, taluk...) பிரிக்குமாறு AI-க்குக் கூடுதல் விதிகள் சேர்க்கப்பட்டன.
  2. `AssistantOverlay.tsx`-இல் மேப்பிங் செய்யப்படும்போது HTML லேபிள்களைக் கொண்டு முகவரி வரிகளைப் பிரிக்கும் வசதி மேம்படுத்தப்பட்டது.
  3. தமிழ்/ஆங்கிலப் புலங்கள் லேபிள்களைக் கொண்டு துல்லியமாக மேப் செய்யப்படுகின்றன.
- **நிலை:** ✅ சரி செய்யப்பட்டது.

### 6. Automation: Document Upload Reliability
- **பிழை:** சில இணையதளங்களில் ஆவணங்கள் பதிவேறாமல் போகுதல்.
- **காரணம்:** `DataTransfer` மாற்றத்திற்குப் பிறகு தேவையான DOM நிகழ்வுகள் (Events) முழுமையாகத் தூண்டப்படவில்லை.
- **தீர்வு:** `automation-logic.cjs`-இல் `change` மற்றும் `input` உடன் சேர்த்து `blur`, `focus` போன்ற கூடுதல் நிகழ்வுகளும் சேர்க்கப்பட்டன.
- **நிலை:** ✅ சரி செய்யப்பட்டது.
